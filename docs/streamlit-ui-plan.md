# Streamlit 월간 캘린더 뷰어 구현 계획

## 요약

병합이 완료된 학교 일정을 확인할 수 있는 작은 Streamlit 앱을 추가합니다.

이 앱은 읽기 전용입니다. 로컬에 저장된 병합 일정 JSON 파일을 읽고, 선택한 월의 일정을 월간 달력과 간단한 목록으로 보여줍니다.

## 주요 변경 사항

- Streamlit을 애플리케이션 의존성에 추가합니다.
- Streamlit 실행 파일을 추가합니다. 권장 위치는 `src/ui/streamlit_app.py`입니다.
- `src/parse/output/merged_events.json` 파일에서 일정을 읽습니다.
- 월간 달력 화면을 렌더링합니다.
- 선택한 월의 일정 목록을 함께 보여줍니다.
- 병합 일정 파일이 없거나, 비어 있거나, 형식이 올바르지 않을 때 이해하기 쉬운 안내 메시지를 보여줍니다.
- README에 Streamlit 실행 명령을 짧게 추가합니다.

## UI 범위

- 사이드바 또는 상단 컨트롤에서는 연도와 월만 선택할 수 있게 합니다.
- 본문에는 다음 두 영역만 표시합니다.
  - 월간 달력
  - 선택한 월의 일정 목록
- 여러 날짜에 걸친 일정은 선택한 월 안에서 해당되는 날짜마다 표시합니다.
- UI에서는 일정 파일이나 캘린더 데이터를 수정하지 않습니다.

## 제외 범위

- Google Calendar API 조회 또는 쓰기
- 이벤트 생성, 수정, 삭제
- 인증 정보 입력 또는 인증 정보 관리 UI
- 검색, 기간 필터, 고급 필터
- 여러 사용자 대상 배포 지원

## 테스트 계획

- 전체 소스에 대해 Python 문법 검사를 실행합니다.
- Streamlit 앱을 로컬에서 실행해 화면이 열리는지 확인합니다.
- 정상적인 병합 일정이 월간 달력에 표시되는지 확인합니다.
- 선택한 월의 일정이 목록에 표시되는지 확인합니다.
- JSON 파일이 없거나, 비어 있거나, 잘못된 형식이어도 앱이 중단되지 않고 안내 메시지를 보여주는지 확인합니다.

## 전제

- 첫 구현은 로컬 실행만 대상으로 합니다.
- 뷰어의 기준 입력 파일은 `src/parse/output/merged_events.json`입니다.
- 일정 데이터는 `title`, `start_date`, `end_date` 값을 가지며 날짜 형식은 `YYYY-MM-DD`입니다.
